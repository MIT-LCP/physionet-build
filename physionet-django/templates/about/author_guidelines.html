Publication Process Overview

These are the steps for publishing a resource on PhysioNet:

  • Create a project.
  • Prepare the project for submission:
    • {% include "about/preparation_checklist.html" %}
  • Submit the project for review.
    1. An editor is assigned and makes a decision:
      1. Reject - The content is not suitable for publication. Submission ends.
      2. Resubmit with revisions - The editor requests changes. The submitting author makes these changes and resubmits.
      3. Accept - The submission continues into the copyedit stage.
    2. The editor copyedits the metadata and files for any final changes. When all authors approve, this editor can complete this step.
    3. The editor publishes the project.

Project Creation

An account is required for creating projects. The project creation page can be found in the navbar under the publish panel, or your project home page, which lists all of your authoring projects. Projects are not accessible by anyone except for their authors.

When creating a project, choose the correct resource type, as this cannot be changed later. If you would like to contribute both data and code from a single related project, submit them in separate projects, and refer them to each other.

Submission and Editing

If the project passes all of its automatic checks, the submitting author may submit the project for review, where an editor will be assigned. While under review, the project cannot be edited.


Editor Decision

The editor will decide whether the project is suitable for publication. The possible decisions are:

  1. Reject - The content is not suitable for publication. Submission ends.
  2. Resubmit - The content is suitable for publication, but changes are required from the submitting author to make it more clear or reusable. Changes may include adding required information, restructuring the files, and rewording content. Once the submitting author has made these changes, they may resubmit the project for review.
  3. Accept - The editor is mostly satisfied with the content. The submission continues into the copyedit stage.

All of the quality assurance guidelines must be met by your submission if it is to be accepted.


Copyedit

The editor makes small changes to the files and metadata that they deem suitable for facilitating reuse of the project. Such changes may include slight rewording of descriptions, renaming of files, and generating a compressed archive of the whole set of files for easy download. When they have finished making changes, they will email the authors. When all authors approve of the changes, the editor may complete this stage.


Publish

The editor publishes the project.
