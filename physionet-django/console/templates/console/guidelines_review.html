Guidelines for reviewers

When reviewing content, check that the requirements below are met. If not, you can copy the relevant points below into your message to the authors, preceded by "Please ensure that:".

All projects

  • A README file is included alongside the files. At minimum, this file should include a title and a brief description of the package content.
  • Sensitive data and protected health information have been removed.
  • Filenames are clear and do not include spaces or special characters (e.g. "/","\,".").

Data (general)

  • All files are in non-proprietary formats (for example, CSV).
  • Tabulated data is arranged such that each variable is a column and each observation (or case) is a row ("tidy data").

Data (waveforms)

  • The shape of waveforms is as expected when plotted as a timeseries.

Software

  • Instructions for installation and usage are clearly documented.
  • Dependencies are clearly indicated, preferably in a requirements file.
  • The software is provided in a non-proprietary language, except where there is a clear reason not to do so (for example, when sharing a Matlab package).
  • Tests are provided to demonstrate correct functioning of major features of the software.
  • The software follows standard style guidelines (for example, PEP8 for Python).
  • The software is logically structured and packaged according to standard practice.

Challenges

  • The objective of the Challenge is clearly stated.
  • A timeline is provided, including a start date, final submission date, and scoring data.
  • The criteria for assessment are clear and unambiguous.
